## 1. 项目概览

- **项目路径**: `f:\pycode\SyaA\syaA`
- **入口文件**: `main.py` → 启动 `src/url/gui_app.py` 中的 `SyaApp`
- **技术栈**: Python + Tkinter (ttk.Notebook) + SQLite
- **数据库**: SQLite，数据库文件 `syaA.db`（项目根目录，可放云盘多机同步）
- **核心特性**: 所有耗时操作在子线程运行，不阻塞 UI；线程安全日志；配置持久化

### 代码结构

| 文件 | 说明 |
|------|------|
| `main.py` | 入口，启动 GUI |
| `src/url/gui_app.py` | **主 GUI 应用**，6个 Tab，约 2100 行 |
| `src/url/getUrl.py` | 旧版命令行工具（功能保留，GUI 版从此提取） |
| `gui_config.json` | 持久化配置文件 |

## 4. Tab 1: 论坛爬取

### 界面布局

| 行 | 元素 |
|----|------|
| 第1行 | 起始帖号 / 结束帖号 / 线程数 |
| 第2行 | 目标网站 URL |
| 第3行 | Cookie文件路径 + 浏览按钮 |
| 第4行 | 开始爬取 / 停止 / 状态文字 / 进度条 / 进度标签 |
| 下方 | 爬取统计面板 + 结果文本区 |

### 输入项说明

| 输入项 | 说明 | 默认值 |
|--------|------|--------|
| 起始帖号 | 爬取范围的起始帖子ID | 3369770 |
| 结束帖号 | 爬取范围的结束帖子ID（不含） | 3380551 |
| 线程数 | 并发线程数，智能调整（不超过帖子数） | 10 |
| 目标网站 | Discuz 论坛域名 | plwt.kpqq4.com |
| Cookie文件 | 浏览器导出的 Cookie 文件路径 | — |

### Cookie 格式

支持两种格式自动识别：
1. **Netscape 格式**（浏览器扩展导出）：`域名 \t 子域 \t 路径 \t 安全 \t 过期 \t key \t value`
2. **旧版 tab 分隔格式**：`key \t value`

### 爬取流程

1. 读取并验证 Cookie（访问一个帖子检查是否被重定向到登录页）
2. 智能分配线程（实际线程数 = min(设定线程数, 帖子总数)）
3. 每个线程处理分配的帖子范围，循环：
   - 构造 URL：`https://{域名}/thread-{帖号}-1-1.html`
   - GET 请求获取页面 HTML
   - 检测登录页（Cookie 失效标志）
   - 提取字段：番号、类型、演员、影片名、大小、码、热度、查看/回复数、磁力链接
   - 写磁力文件：`./output/magnet/{日期}-magnet.txt`（排除黑名单和-UC）
   - 检查数据库是否已存在
   - 保存源文件：`./output/高清中文字幕网页源文件/{番号}.txt`
   - 入库 INSERT
4. 实时更新进度条：`已处理/总数 (百分比%)`

### 字段提取逻辑

使用 `extract_field()` 函数，三策略逐级降级：

1. **策略1**：从 `<font>` 标签区域提取（最可靠，值内可包含【】）
   - 匹配 `<font...>【字段名】：值</font>`
2. **策略2**：从 meta description 提取（以下一个已知字段名作为终止）
   - 允许值内部包含【】
3. **策略3**：简单兜底，遇到 `<` 或 `"` 停止

### Cookie 失效检测

- 连续遇到登录页时累加计数
- 连续 3 次判定为 Cookie 失效，自动终止所有线程
- 终止时汇报：终止原因 + 已处理进度统计

### 爬取统计

| 指标 | 含义 |
|------|------|
| new | 新入库数 |
| exist | 数据库已存在数 |
| non_cn | 非中文字幕跳过数 |
| no_magnet | 无磁力链接跳过数 |
| fail | 处理失败数 |
| other | 其他异常数 |

| 排序方式 | 随机 / 热度降序 / 查看降序 / 最新入库 | 随机 |
| 仅白名单演员 | 勾选后只返回白名单演员的磁力 | 未勾选 |

| 演员搜索 | 按演员名模糊匹配（LIKE %关键词%） | 空 |
| 类型筛选 | 全部 / 有码 / 无码 | 全部 |
| 最小大小 | 过滤小于指定 GB 的文件 | 0 |

### 按钮功能

| 按钮 | 说明 |
|------|------|
| **生成磁力** | 按筛选条件查询数据库，生成磁力链接列表 |
| **全部白名单磁力** | 遍历数据库所有记录，只返回白名单演员的磁力（无视生成数量和排序） |
| **复制磁力链接** | 只复制纯磁力链接到剪贴板（不含影片信息行） |
| **保存到文件** | 弹窗选择保存路径，默认文件名 `magnet-日期.txt` |
| **清空** | 清空结果文本区 |

### SQL 查询逻辑

筛选条件下推到 SQL 层：
- `local_path IS NULL` — 未下载
- `grade IS NULL` — 未评分
- `magnet IS NOT NULL` — 有磁力链接
- `numbers_name NOT LIKE '%-UC'` — 排除无码破解版
- 有码/无码筛选：`mosaic = '有码'` / `mosaic = '无码'`
- 演员搜索：`performer LIKE '%关键词%'`
- 最小大小：`CAST(REGEXP_REPLACE(size, '[^0-9.]', '') AS DECIMAL) >= 阈值`

Python 层额外过滤：
- 黑名单演员：始终排除
- 白名单演员：勾选"仅白名单"时排除非白名单

### 统计输出

```
[磁力] 查询到 80 条，黑名单过滤 3 条，非白名单过滤 15 条，最终 62 条
```

## 7. Tab 4: 数据库管理

### 界面布局

按钮行1 + 按钮行2(JSON/迁移) + 115路径输入 + 结果文本区

### 功能按钮

| 按钮 | 说明 |
|------|------|
| **刷新115文件到数据库** | 递归扫描指定115路径，按番号匹配更新 `exist_in_115` 和 `local_path` |
| **校验番号格式** | 遍历全表，用正则规范化番号（如 `ABC-123`），更新 `designation` 字段 |
| **清理无效路径** | 遍历有 `local_path` 的记录，文件不存在则置空路径和115标记 |
| **去重无码版本** | 查所有无码记录，若同一番号(designation)有多条，移除无码版本的路径 |
| **外部视频入库** | 扫描指定路径下未入库的视频文件，自动编号为 `zzz-N` 并 INSERT |
| **查看统计** | 显示各类计数统计 |
| **导出JSON** | 将 av、black_table、good_performer 三张表导出为 JSON 文件（备份/分享/跨机同步） |
| **导入JSON** | 从 JSON 文件导入数据到 SQLite，已存在的记录自动跳过（INSERT OR IGNORE） |
| **从MySQL迁移** | 一键从 MySQL(h_db) 迁移全部数据到当前 SQLite 数据库（首次迁移用） |

### 统计内容

| 统计项 | SQL |
|--------|-----|
| 总记录数 | `SELECT count(*) FROM av` |
| 有本地路径 | `WHERE local_path IS NOT NULL` |
| 未评分 | `WHERE grade IS NULL` |
| 有磁力链接 | `WHERE magnet IS NOT NULL` |
| 黑名单演员 | `SELECT count(*) FROM black_table` |
| 关注演员 | `SELECT count(*) FROM good_performer` |
| 有码/无码 | `WHERE mosaic = '有码'/'无码'` |

## 8. Tab 5: 文件整理

### 界面布局

| 行 | 元素 |
|----|------|
| 第1行 | 操作路径 + 浏览按钮 |
| 第2行 | 删除垃圾文件 / 删除空文件夹 / 小文件阈值(MB) + 删除小文件 |
| 第3行 | 重命名视频 / 目标路径 + 搬运视频 |
| 第4行 | 删除黑名单演员视频 / 低分阈值 + 删除低分视频 / 关键词 + 关键词删除 |
| 第5行 | 一键整理流水线 |
| 第6行 | 高分阈值 + 收集目标路径 + 浏览 + 高分收集 |

### 功能详解

#### ① 删除垃圾文件
- 递归遍历"操作路径"下所有文件
- **跳过 WHITE_LIST 文件夹**（电脑文件备份、本地照片视频库、手机备份、手机相册）
- 删除条件：
  - 文件 < 50MB → 直接删
  - 文件名匹配 `DEL_FILE_NAME_LIST`（广告/引流文件）→ 删
  - 空文件夹 → 删

#### ② 删除空文件夹
- 递归删除所有空目录（跳过 WHITE_LIST）

#### ③ 删除小文件
- 删除小于设定阈值（默认50MB）的文件
- 只删文件，不删文件夹

#### ④ 重命名视频
- 从文件名用正则提取番号（如 `ABC-123`）
- 查数据库匹配，重命名为 `热度-演员-片名-大小.mp4` 格式

#### ⑤ 搬运视频
- 将"操作路径"下所有 `.mp4` 文件移动到"目标路径"
- 递归处理子目录

#### ⑥ 删除黑名单演员视频
- **不依赖操作路径**，直接查数据库 `black_table`
- **只删除独演影片**：performer 字段清理 HTML 残留后必须完全等于黑名单演员名
- 多演员影片（如 `辻かのん 楓カレン`）**不会被删除**——需手动用"关键词删除"处理
- **白名单保护**：如果该演员同时在白名单中，提示黑白名单冲突并跳过删除；若黑名单名字含 HTML 残留（如 `&nbsp;`）与白名单实际为同一人，自动从黑名单移除脏数据
- **高分保护**：如果该演员有评分 ≥ 90 的独演视频，列出所有高分影片并跳过删除，让你判断
- 自动清理 performer 字段中的 HTML 残留（`&nbsp;`、`</td>` 等）
- 日志会逐个输出正在检查的演员名

#### ⑦ 删除低分视频
- 查数据库评分 ≤ 阈值（默认50）且有本地路径的记录
- 删除文件 + 清空路径记录

#### ⑧ 关键词删除
- 按影片名称模糊匹配关键词（`LIKE %关键词%`）
- 删除匹配的本地文件 + 清空路径记录

#### ⑨ 一键整理流水线

按顺序执行6步：

```
步骤1: 删除垃圾文件
步骤2: 重命名视频
步骤3: 搬运视频（操作路径 → 目标路径）
步骤4: 清理空文件夹
步骤5: 录入115（扫描目标路径更新数据库）
步骤6: 删除黑名单演员视频
```

#### ⑩ 高分收集

将评分 ≥ 设定阈值（默认90）的视频**复制**到指定目录，**按分数分子目录**存放，同时生成 HTML 目录清单和 M3U 播放列表。

**⚠ 预览机制：点击"高分收集"后会先弹出预览窗口，列出所有将执行的动作（同盘移动/跨盘复制/评分变更/降级移回/跳过），确认后才真正执行。**

**输入项：**

| 输入项 | 说明 | 默认值 |
|--------|------|--------|
| 高分阈值 | 评分 ≥ 此值的视频会被收集 | 90 |
| 收集目标路径 | 复制到的目标目录 | 空 |

**预览窗口说明：**
- 顶部摘要：显示阈值、目标路径、各类型操作数量
- 中间列表：每条操作的类型/番号/评分/说明/来源/目标
  - 🟠 同盘移动：源文件移走，DB路径更新
  - 🔵 跨盘复制：源文件保留
  - 🟢 评分变更：子目录间移动
  - 🔴 降级移回：移回原始目录
  - ⚪ 跳过：已存在或文件不存在
- 底部按钮："确认执行" / "取消"

**目录结构：**
```
收集目录/
  90分/
    CAWD-524-美城露露-有码-xxx.mp4
  95分/
    ABC-123-xxx-有码-xxx.mp4
  100分/
    XYZ-999-xxx-有码-xxx.mp4
  highgrade_catalog.html
  highgrade_playlist.m3u
```

**功能流程（3阶段）：**

1. **阶段1：同步旧文件** — 扫描目标目录下已有的视频，用数据库 `local_path` 字段精确匹配，对比最新评分
   - 评分变更的视频自动移动到正确子目录（如 90分→95分），并更新 DB 路径
   - 评分降至阈值以下的视频**移回原始目录**（而非删除），并更新 DB 路径
   - 清理空的子目录
2. **阶段2：复制/移动新文件** — 查询数据库中评分 ≥ 阈值且有本地路径的记录
   - **同盘（如115网盘Z:内）**：使用 `shutil.move`（服务端操作，秒完成），源目录文件会移到目标子目录，数据库 `local_path` 自动更新
   - **跨盘**：使用 `shutil.copy2`（本地IO，较慢但保留源文件）
   - 按评分分子目录（`dst_path/XX分/filename.mp4`）
   - 已存在且文件大小相同的自动跳过
   - 源文件不存在的记录跳过并记录日志
   - 日志含：进度 `[3/115]`、文件大小、耗时、速度（MB/s）
3. **阶段3：生成报告**
   - **HTML 目录清单**（`highgrade_catalog.html`）：暗色主题，按评分分组，显示番号/演员/片名/码制
   - **M3U 播放列表**（`highgrade_playlist.m3u`）：含子目录的相对路径，可用播放器直接加载

**增量更新说明：**
- 多次运行安全：已存在的文件不会重复复制
- 评分变更自动同步：上次在90分目录的视频，评分改成95 → 自动移到95分目录
- 评分降低自动移回：评分降到阈值以下 → 移回原始目录（**不会删除文件**）

### 典型使用流程

1. 下载完成后，操作路径指向云下载目录
2. 先点 **"删除垃圾文件"** 清理广告和碎片
3. 点 **"重命名视频"** 规范化文件名
4. 设好目标路径，点 **"搬运视频"** 移到归档目录
5. 点 **"删除空文件夹"** 清理残留
6. 最后可点 **"删除黑名单演员视频"** 清理

或者一步到位：**"一键整理流水线"**

### 功能说明

采用**三步下载策略**，利用站内TXT直链高效下载：

**步骤1 — 访问详情页获取元数据**
- URL格式：`{根URL}{ID}/txt.html`
- 提取书名、作者、所有分卷的下载链接列表
- 详情页请求也带3次重试

**步骤2 — 逐卷下载TXT直链**
- 分卷URL格式：`{根URL}txt/?id={ID}&p={卷号}`
- 每个分卷对应一个 `.txt` 文件（如 `捞尸人(1-500章).txt`）
- 每卷独立下载，带3次重试 + 详细耗时/大小日志 + 实时进度（每10秒汇报速度和大小）
- 使用**流式下载（stream=True）**，分块读取避免大文件IncompleteRead
- 单卷失败不影响其他卷

**步骤3 — 合并为整本**
- 所有分卷合并为一个完整文件
- 自动添加元信息头（书名、作者、来源、时间）
- 分卷之间用分隔线标记
- 保存到 `./output/book/{日期}/{ID}-{书名}-{作者}.txt`

### 增量更新（更新小说按钮）

点击「**更新小说**」进入增量模式，**基于本地 `book_index.json` 索引自动更新所有已下载的小说**，无需输入起止ID：

**工作流程：**
1. 读取本地索引 → 获取所有已记录小说列表
2. 逐本访问详情页 → 检测是否有新分卷/新章节
3. 有新内容则下载并追加，已是最新则跳过
4. 完成后输出汇总统计

**混合策略：**
```
首次下载 → TXT分卷直链（快，整本拿下）
           ↓ 记录 latest_chapter_num 到索引

点击"更新小说"：
  ① 读 book_index.json → 列出所有已下载的书
  ② 逐本拉详情页 → 提取当前最新章节号
  ③ 与索引中 latest_chapter_num 比较
     - 相同 → 真正最新，秒跳过 ✓
     - 更大 → 有新章节！
  ④ 分卷级：新分卷？→ 用TXT直链下载（快）
  ⑤ 章节级：旧分卷中有新章节？→ 逐章下载并追加到文件末尾 ⭐
  ⑥ 更新索引
```

**章节级检测原理：**
- 站点提供每章独立页面：`/{book_id}/{文章ID}.html`（纯文本~1000字/章）
- 目录页 `/{book_id}/` 列出全部章节（含URL和章节号），可分页浏览
- 详情页可解析出当前最新章节号

**索引文件格式（book_index.json）：**
```json
{
  "2": {
    "id": 2,
    "name": "苟在初圣魔门当人材",
    "author": "鹤守月满池",
    "file_path": "output\\book\\20260505\\2-xxx.txt",
    "total_parts": 4,
    "downloaded_parts": [1, 2, 3, 4],
    "latest_chapter_num": 1449,
    "is_completed": false,
    "last_update": "2026-05-05 15:34:37",
    "detail_url": "https://www.sudugu.org/2/txt.html"
  }
}
```

**字段说明：**
| 字段 | 类型 | 说明 |
|------|------|------|
| `is_completed` | bool | **是否已完结**。`true`=完结小说，更新时自动跳过；`false`=连载中，正常检查更新 |

**更新策略 — 完结过滤：**
- 点击「更新小说」→ 读本地索引 → **逐本访问详情页**
- 检测页面是否含"完结"字样 → 写入 `is_completed` 字段
- **已完结的小说直接跳过**，不再浪费请求检测分卷/章节变化
- 只对 `is_completed=false` 的连载中小说执行增量检测（分卷级 + 章节级）
- 日志输出：`✓已完结` / `连载中`

### 全量爬取的智能增量（⭐自动检测）

点击「开始爬取」遍历 ID 范围时，工具会**自动识别每本书是否已经下载过**：

```
全量爬取流程：
  遍历 id=1~100
  ├── id=1 → 索引中无记录 → 新书，全量下载全部分卷
  ├── id=2 → 索引有记录 + 连载中 → 自动走更新流程（只下新卷+新章）
  ├── id=3 → 索引有记录 + 已完结 → 跳过
  └── id=5 → 索引无记录 → 新书，全量下载...
```

**行为对比：**

| 场景 | 书在索引中？ | 状态 | 行为 |
|------|:---:|:---:|------|
| 全量爬取 | ❌ | - | 全量下载所有分卷 |
| 全量爬取 | ✅ | 连载中 | **自动增量**（跳过已有卷 + 章节追加） |
| 全量爬取 | ✅ | 已完结 | 跳过 |
| 更新按钮 | ✅ | 连载中 | 增量更新 |
| 更新按钮 | ✅ | 已完结 | 跳过 |

这意味着：**全量爬取 = 新书下载 + 已有书的自动更新合并在一起**，无需手动区分。

### 全站爬取（⭐一键爬全站）

点击「**全站爬取**」按钮，自动完成两阶段操作：

```
第一阶段：采集列表
  请求 /zuixin/1.html → 获取总页数（如408页）
  遍历 /zuixin/1~408.html → 提取每页所有书籍ID（去重）
  输出：有序书籍ID列表 [2, 3, 4, 5, 21, ...]

第二阶段：逐本处理
  对每本书执行智能判断：
    NEW     → 索引中无记录   → 全量下载
    已知/连载 → 有新内容      → 增量追加
    已知/最新 → 无新内容       → 秒跳
    已知/完结 →               → 秒跳
```

**UI布局变化：**
| 第2行 | ... | **开始爬取** | **全站爬取** | **更新小说** | ... |

**优化点：**
- 列表采集时每10页汇报一次进度，实时可见
- 采集完成后显示本地已记录数 vs 新书数对比
- 第二阶段复用完整的双层增量逻辑（分卷+章节）
- 三个操作按钮互斥，运行时任一点另一个都禁用

**适用场景：**
- ✅ **勾选「跳过已有卷」（默认）**：
  - 作者新出了分卷 → 只下新分卷并追加
  - 已有分卷中追更了新章节 → 自动检测并逐章追加（⭐章节级增量）
  - 上次部分失败 → 自动补漏
- ❌ **取消「跳过已有卷」**：强制重新下载所有卷（覆盖旧文件）

### 可靠性机制

- **3次重试**：详情页 + 每个分卷均支持3次自动重试（间隔2秒）
- **停止按钮**：随时中断，已下载的分卷会保留并合并（不会丢失进度）
- **进度条**：ID粒度显示处理进度
- **实时下载进度**：每~10秒输出已下大小(KB)、当前速度(KB/s)、耗时(s)
- **流式下载**：使用 stream=True 分块读取，避免大文件连接中断
- **详细日志**：每步都有URL、耗时(ms)、大小(字节)、错误信息
- **爬取统计**：完成后输出成功/空内容/失败/**已是最新(跳过)** 数量汇总
- **章节级增量更新**（⭐新）：检测到已有分卷内的新章节时，自动逐章下载并追加到文件末尾，无需重下整卷

## 12. 注意事项与已知问题

### 注意事项

1. **Cookie 时效性**：Cookie 会过期，爬取前务必确认 Cookie 有效；工具会在爬取前自动验证
2. **数据库编码**：Unicode 编码异常时会自动降级处理（移除非 ASCII 字符），异常影片源文件会保存到 `./output/异常影片/`
3. **线程安全**：日志通过队列传递到主线程；进度条通过 `root.after` 更新；Cookie 失效计数通过 Lock 保护
4. **SQLite 同步**：数据库文件 `syaA.db` 放在云盘目录即可多机同步；注意不要两台电脑同时写入
5. **字段提取**：影片名称值内部可能包含【】，已用多策略提取函数处理

### 已知问题

- 重命名功能依赖文件名中包含标准番号格式，非标准命名的文件会被跳过
- 搬运功能只处理 `.mp4` 文件，其他格式不会搬运
- 外部视频入库时按 zzz 递增编号，删除中间记录后编号不连续（不影响功能）
