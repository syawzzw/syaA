## 封面缩略图改版（2026-05-26 晚）
- **用户反馈**：封面仍无法显示，且文字信息区被挤压显示不全
- **根因分析**：
  1. 封面 Canvas 是独立大区域（320x200），占用了文字信息区的空间
  2. OpenCV 未安装，无法从视频文件提取真实缩略图
  3. Windows Shell API 缩略图提取（ctypes + IShellItemImageFactory）因 COM 调用约定问题失败
- **改造方案**：
  1. **布局改造**：封面从独立 LabelFrame 改为右上角浮动小缩略图（160x90），使用 `place(relx=1.0, rely=0.0)` 叠加在文字信息区上方，不再挤压文字

  4. **文字区留白**：`_update_video_info` 开头插入 5 行空行，避免文字被缩略图遮挡
- **技术细节**：
  - 缩略图文件名用番号（替换非法字符），避免原文件名含日文/中文导致路径问题
  - `_extract_video_thumbnail` 增加帧号越界保护（超过总帧数时取中间帧）
  - `_display_pil_image` 和 `_display_image_file` 都缩放到 `self._thumb_w` x `self._thumb_h`
  - `_toggle_theme` 增加对 `cover_canvas` 的边框颜色更新

## 修复 SQLite "database is locked"（2026-05-26 深夜续）
- **问题**：一键整理流水线步骤5（录入115）时，每个文件单独 `get_db_conn()`→`link_db_cmd()`→close，高频写入+网络盘IO延迟导致锁竞争，42次 locked 错误
- **根因**：MySQL 没有这个因为它是服务端锁；SQLite 是文件锁，并发写入（每条记录单独开连接+commit）在IO慢的盘上冲突严重
- **修复**：
  1. `get_db_conn(timeout=30)`：SQLite 遇锁自动等待最多30秒（默认5秒太短）
  2. `link_db_cmd` 加 retries=3 递增等待重试
  3. 批量操作改为复用同一连接+事务：`_check_115_recursive(path, con)`、`_rename_recursive(path, con)`
  4. `_delete_black_worker`、`_delete_lowgrade_worker`、`_delete_keyword_worker` 内部 `link_db_cmd` 改为 `con.cursor().execute()`
  5. 115扫描每100条 commit 一次
  6. 流水线 `_pipeline_worker` 步骤2和5分别创建连接
- 文件修改：`src/url/gui_app.py`、`GUIDE.md`
