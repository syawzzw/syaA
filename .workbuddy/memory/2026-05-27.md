- 3阶段流程：
  1. 同步旧文件：用DB的local_path精确匹配（而非从文件名猜番号），评分变更的移到正确子目录，降至阈值的**移回原始目录**（不再删除！）
  2. 复制/移动新文件：**同盘用 shutil.move（115网盘服务端操作，秒完成，源文件移走并更新DB路径）**，跨盘用 shutil.copy2
  3. 生成报告：HTML 目录清单 + M3U 播放列表

- **115网盘特性发现**：
  - `shutil.move` 同盘 = 服务端操作（6GB文件0.3秒）
  - `shutil.copy2` 同盘 = 本地IO（卡死）
  - `os.link` 硬链接 = 不支持（WinError 50）

## 高分收集阶段1逻辑修复
- **问题**：用90分阈值收集时，扫描到80分/目录下的ADN-425（评分80），因为 80 < 90，被判定为"降级移回"，但文件本来就在正确位置
- **根因**：阶段1的判断条件是 `new_grade < grade`（评分低于当前阈值就移回），这是错误的——80分的文件在80分目录下是正确的，不管当前阈值是多少
- **修复**：改为"子目录分数与DB评分一致 → 跳过"的逻辑
  1. 提取子目录名中的分数（如 `80分` → 80）
  2. 子目录分数 == DB评分 → 文件在正确位置，完全不动
  3. DB评分变更但 ≥ 阈值 → 移到正确子目录
  4. DB评分比子目录分数低（真正降级，如之前90分现在80分）→ 移回原目录
  5. 其他不匹配情况 → 移到正确子目录
- 预览 `_collect_preview_worker` 和执行 `_collect_highgrade_worker` 两处都已修复

## 同番号多条记录时的阶段1修复
- **问题**：SONE-153在DB中有两条记录（80分和90分），同盘move后两条的local_path都指向90分/目录。阶段1扫描时匹配到80分那条，发现80 != 90，想移到80分/目录
- **修复**：在"子目录分数==DB评分一致→跳过"之后，增加"同番号多记录检查"：
  - 查询 `SELECT COUNT(*) FROM av WHERE local_path = ? AND grade = ?`（grade=子目录分数）
  - 如果有评分==子目录分数的记录也指向此文件 → 文件位置正确，不动
- 预览和执行两处都加了此检查

## av表新增 created_at / updated_at 字段
- 新增两个字段：`created_at TEXT`（存入时间）、`updated_at TEXT`（更新时间）
- **created_at**：INSERT 时写入当前时间，历史记录为空
- **updated_at**：每次 UPDATE 时自动更新当前时间
- **自动更新机制**：
  - `link_db_cmd` 内部调用 `_append_updated_at()` 自动追加
  - 直接用 cursor 执行的 UPDATE 手动调用 `_append_updated_at()` 处理（7处）
- `_append_updated_at(sql, val)` 辅助函数：对 UPDATE av 语句在 WHERE 前插入 `, updated_at = ?`
- AV_COLUMNS 新增 `created_at, updated_at`；IDX_CREATED_AT=15, IDX_UPDATED_AT=16
- 5处 cur_video 赋值都加了 created_at/updated_at 读取
- `_update_video_info` 显示"存入"和"更新"时间
- 建表语句新增两字段 + ALTER TABLE 为旧数据库兼容
- 两处 INSERT 都加入 created_at/updated_at
