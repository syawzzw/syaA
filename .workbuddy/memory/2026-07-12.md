# 2026-07-12 工作日志

## performer 字段 HTML 溢出 Bug 修复
- **问题**: `_extract_field()` 正则贪婪匹配导致 performer 字段被后续字段内容污染
- **典型例子**: SNOS-227 的 performer 从期望的 `三田真鈴` 变成 79 字符含 `【影片容...98堂[原色花堂]" />` 的溢出文本
- **根因**: 论坛 HTML 结构变化后，策略2/3的正则把后续字段（影片容量、番号、影片名称等）吃进 performer
- **修复方案**: 新增 `SyaApp._clean_performer_overflow()` 静态方法，5层防御机制：
  - Layer0: 短字符串(≤30字符)直接放行
  - Layer1: HTML属性残留检测（`" />` 等尾部标记→回溯截断）
  - Layer2: 字段溢出检测（`【影片容`、`[无码破解]`、`98堂`等标记→截断）
  - Layer3: 方括号溢出处理（非法`【`无配对`】`→截断）
  - Layer4: 兜底截断（空格边界优先，硬截断到30字符）
- **修改文件**:
  - `src/url/gui_app.py`: 新增方法 + 普通爬取(L1875)和离线爬取(L2906)两处调用
  - `fix_performer_overflow.py`: 数据库清洗脚本（新建，预览+execute双模式）
- **影响范围**: 212条 performer>30字符的记录需清洗，预览验证全部可自动修复
- **注意**: 脚本使用了 `av.id` 列但实际主键是 `numbers_name`，已修复脚本中的列名引用
