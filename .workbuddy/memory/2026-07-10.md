# 2026-07-10 工作日志

## 离线爬取 Cookie 失效检测漏洞修复

### 问题
- "其他"分区下 16987 条帖子，其中 4049 条是"提示信息"页面（Cookie 失效后论坛返回的错误页）
- 论坛"提示信息"页面 8376~11122 字节，包含 `member.php?mod=logging`/`action=login`/`登录` 等 Cookie 失效特征
- 但 `_is_login_page()` 的长度阈值逻辑（<2000 检查、<5000 检查）导致 >5000 字节的提示信息页绕过了检测
- 普通爬取因为额外检查磁力链接，提示信息页会被 `no_magnet` 跳过；离线爬取不检查磁力链接，直接保存了整个错误页

### 修复内容
1. **`_is_login_page()` 重写**（L1713）：
   - 去掉长度限制，改为：先检查是否有 `postlist`/`id="post_"`/`ajaxdialog`（正常帖子特征）→ 有则直接返回 False
   - 没有正常帖子特征 → 不限页面大小，直接检查登录特征词
   - 新增 `<title>提示信息</title>` 检测（Discuz Cookie 失效提示页）
   - 新增 502/503 服务器错误页检测
   - 移除了原来 login_signs 中的 `"登录"`（太宽泛，正常帖子页面也可能包含），改为更精确的 `"请先登录"`/`"您需要登录"`

2. **分区提取 fallback 修复**（L2153）：
   - 默认值从 `"其他"` 改为 `"unknown"`
   - 避免分区面包屑提取失败时误入白名单导致非白名单帖子被保存

### 影响范围
- 修复同时惠及普通爬取和离线爬取（共用 `_is_login_page()`）
- 非白名单分区的 2421 条已保存帖子（AI专区821、转帖交流区371等）后续不会再被错误保存
