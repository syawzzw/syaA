## 搜索栏JS语法Bug修复
- **问题**：搜索 JS 中 `+'</div>'` 和 `if(...)` 之间缺少分号，导致 `+'</div>'if(...)` 被解析为语法错误
  - 具体位置：`h+=...+'</div>'if(hits[i].excerpt){...}` — JS 引擎把字符串后面紧跟 if 语句当成语法错误
  - 同理 `if` 块结尾 `}` 后紧跟 `+` 运算符也有问题
- **修复**：把每个 `h+=` 改为独立语句（加分号），`if` 块也独立
  - 旧：`h+='..'</div>'if(...){h+='..';}+'<div>'`
  - 新：`h+='..'</div>';if(...){h+='..';}h+='<div>';`
- 两处都需要修：增量更新 `_update_main_index`（行2829）和全量 `_generate_offline_index`（行3083）
- `replace_all=true` 一次修复两处
