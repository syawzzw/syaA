## 关键 Bug 修复：评分功能数据库不更新
- **根因**：`_append_auto_fields()` 在 SQL 的 SET 子句插入了 `updated_at=?` 和 `last_action=?` 占位符，但参数用 `list(val) + add_vals` 追加到末尾，导致参数顺序与占位符错位
- **现象**：UPDATE av SET grade=?, updated_at=?, last_action=? WHERE numbers_name=? 绑定参数变成了 (10, 'zzz-1215', 时间戳, '评分')，WHERE 条件匹配不到任何记录，rowcount=0
- **修复**：改为根据 SET 部分占位符数量，将新增参数插到 SET 参数之后、WHERE 参数之前：`val_list[:set_count] + add_vals + val_list[set_count:]`
- **涉及文件**：src/url/gui_app.py 的 `_append_auto_fields()` 函数（第250行附近）
