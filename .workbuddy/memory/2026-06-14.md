## 离线爬取"用户中断"误报 Bug 修复
- **现象**：离线爬取自动终止，日志显示"爬取被用户手动停止"，但用户没有点停止按钮
- **根因1**：`_crawl_verify_post_id` 从未被赋值，始终为0；当遇到短页面触发Cookie二次验证时，`if not test_post_id: return True` 永远判定Cookie失效，导致 `crawl_running=False`
- **根因2**：离线爬取完成回调只检查 `crawl_running`，不检查 `_crawl_stop_reason`，Cookie失效也显示"用户手动停止"
- **根因3**：`_crawl_stop_reason` 在两次爬取间未重置，残留上次状态
- **修复**：
  - 启动爬取时设置 `_crawl_verify_post_id = begin`（普通+离线两处）
  - 启动爬取时重置 `_crawl_stop_reason = ""`（普通+离线两处）
  - 离线爬取完成回调检查 `_crawl_stop_reason`，区分"Cookie失效"和"用户手动停止"
